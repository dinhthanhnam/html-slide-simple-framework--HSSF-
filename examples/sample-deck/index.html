01

Vì sao cần Git?

© 2026 By Rikkei Academy - Rikkei Education - All rights reserved.

Vấn đề khi không có VCS

So sánh thực tế fresher

Hạn chế

  • Copy folder: project_final_v2
  • Mất code / ghi đè lẫn nhau
  • Khó biết ai sửa gì, khi nào

Có Git

  • Lịch sử rõ ràng, rollback được
  • Làm việc song song an toàn
  • Review & hợp tác qua remote

Mẹo

Tạo branch, commit, merge

Flow chuẩn fresher

  1. 01

    Tạo & checkout

    git switch -c feature/x

  2. 02

    Commit trên feature

    add → commit message rõ

  3. 03

    Merge về main

    switch main → merge feature

branch.sh bash 
git switch -c feature/validate-email
# ... code + commit ...
git switch main
git merge feature/validate-email
© 2026 By Rikkei Academy - Rikkei Education - All rights reserved.

Những việc không nên làm

Anti-patterns fresher

  • Force push lên main chung của team
  • Commit file .env, secret, file build lớn
  • Message kiểu “fix”, “update”, “asdasd”
  • Sửa trực tiếp trên main khi đang có feature dở

Lưu ý

git push --force trên nhánh shared có thể xóa lịch sử đồng đội. Hỏi mentor trước.

Lab: clone → branch → PR

Thực hành 20 phút

lab.sh bash 
git clone <repo-url>
cd project
git switch -c feature/your-name
# sửa README → add → commit
git push -u origin HEAD
# mở Pull Request trên GitHub/GitLab

Tổng kết

Mang về áp dụng ngay

  • Git lưu lịch sử — không còn “folder final final”
  • Ba vùng: working → staging → repository
  • Branch để làm việc song song an toàn
  • Message rõ = team review nhanh hơn
